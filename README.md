# ITP Charge Mapper

A lightweight Python utility for safely replacing atomic partial charges
in **GROMACS `.itp` topology files** using charges obtained from
**quantum chemistry calculations** (such as **RESP / RESP2**).

In many molecular dynamics workflows, partial charges derived from
quantum chemical calculations are exported as simple coordinate--charge
files (for example `.chg`). Meanwhile, topology files used in
simulations are often generated by external parameterization tools such
as **SwissParam**, **CGenFF**, or similar utilities. These tools may
reorder atoms when constructing the topology, which makes direct manual
replacement of charges error‑prone. This script provides a reproducible
and automated solution to correctly map those charges onto the topology.

------------------------------------------------------------------------

## Features

This script automatically maps quantum‑chemistry‑derived charges to
atoms in a GROMACS topology file by using atom names from a reference
MOL2 structure as the mapping bridge. It preserves all original topology
parameters such as bonded terms, atom types, and masses while modifying
only the charge column in the `[ atoms ]` section. Multiple safety
checks are implemented, including verification of atom counts,
uniqueness of atom names in the reference structure, and completeness of
the charge mapping. The script reports the total charge after
replacement and optionally enforces a target total charge (e.g., +1 for
a cation). When enforcing a target total charge, a small numerical
correction can be applied automatically to non‑critical atoms such as
hydrogens. A configurable threshold protection mechanism ensures that
the automatic correction is only applied when the deviation is within a
safe numerical tolerance, preventing accidental modification of
chemically meaningful charge distributions.

------------------------------------------------------------------------

## Workflow

The script assumes the following relationship between files:

    Charge output (.chg)
            │
            │ same atom order
            ▼
    reference structure (.mol2)
            │
            │ atom name mapping
            ▼
    GROMACS topology (.itp)

The `.mol2` file acts as the **reference for atom names**, enabling
reliable mapping even if the `.itp` topology has a different atom
ordering.

------------------------------------------------------------------------

## Usage

Basic usage:

``` bash
python replace_itp_charges.py \
  --mmff-mol2 "structure.mol2" \
  --chg "resp2.chg" \
  --itp-in "molecule.itp" \
  --itp-out "molecule_resp.itp"
```

Optional parameters:

    --decimals N

Number of decimal places written to the output topology (default: 8).

    --target-total-charge Q

Force the final topology to have an exact total charge `Q`. A tiny
correction will be applied to selected atoms if needed.

    --adjust-on-elements H

Preferred element types to receive the correction (commonly hydrogen
atoms).

    --adjust-on-atoms H12 H15

Explicit atom names that should receive the correction.

    --max-correction-abs VALUE

Safety threshold for automatic charge correction. If the required
correction exceeds this value, the script will stop with an error
instead of modifying the topology.\
Default: `1e-4`

------------------------------------------------------------------------

## Input Files

### Charge file (`.chg`)

Typical format:

    Element   x   y   z   charge
    C         ... ... ...  -0.12345

The order of atoms must match the reference `.mol2` structure.

### Reference MOL2 structure

Provides **unique atom names** that allow the script to correctly map
charges to the topology.

### GROMACS ITP topology

The script modifies only the **charge column in the `[ atoms ]`
section**.\
All other force‑field parameters remain unchanged.

------------------------------------------------------------------------

## Output

The script generates a modified `.itp` file with updated charges and
prints diagnostic information including:

-   number of atoms processed
-   number of charges replaced
-   total charge of the resulting topology
-   applied correction (if target charge enforcement is enabled)

------------------------------------------------------------------------

## Safety Notes

The script assumes:

-   atom counts are identical across files
-   atom names in the `.mol2` reference structure are **unique**
-   the `.chg` file follows the same atom ordering as the `.mol2`
    structure

Users should always verify the resulting topology before running
production molecular dynamics simulations.

------------------------------------------------------------------------

## Motivation

Combining **quantum‑derived electrostatic charges** with **classical
force‑field parameters** is common in molecular simulations. However,
atom reordering introduced by topology generation tools often makes
manual charge replacement unreliable. This tool aims to provide a simple
and reproducible step to integrate RESP‑style charges into GROMACS
topologies safely.
